# 版本管理说明

## 版本号怎么改

版本号使用 `主版本.次版本.修订版本`，例如 `1.0.0`。

- 修订版本：只修 bug 或小优化，例如 `1.0.1`。
- 次版本：新增功能但不破坏原有功能，例如 `1.1.0`。
- 主版本：大改版或不兼容旧版本，例如 `2.0.0`。

## 每次发布前做什么

1. 修改根目录 `VERSION`。
2. 修改 `frontend/package.json` 的 `version`。
3. 修改 `backend/package.json` 的 `version`。
4. 在 `CHANGELOG.md` 顶部新增一段版本记录。
5. 跑前端构建和后端测试。
6. 备份数据库和上传目录。
7. 部署到服务器。

## 推荐示例

如果只是修复登录超时问题：

- `VERSION` 从 `1.0.0` 改成 `1.0.1`
- `CHANGELOG.md` 新增：

```md
## v1.0.1 - 2026-06-21

### 修复

- 调整登录有效期，减少频繁重新登录。
```

如果新增了一个“客户管理”模块：

- `VERSION` 从 `1.0.1` 改成 `1.1.0`
- `CHANGELOG.md` 记录新功能和影响范围。

## 上线后怎么回退

上线前保留三样东西：

- 当前代码版本
- 当前数据库备份
- 当前上传目录备份

如果上线后出问题，先回退代码版本，再恢复对应数据库备份。
